## üéØ Project Overview

The Sales Data Analysis project helps understand:

- Sales performance  
- Profit trends  
- Customer and product behavior

- Time-based analysis  

Using Power BI, raw data is cleaned, modeled, and visualized to support decision-making.

## üîÑ 2. Data Transformation

Performed data cleaning and transformation using **Power Query Editor**:

- Removed duplicates and null values.  
- Renamed columns for clarity.  
- Changed data types.  
- Created calculated columns.  
- Split and merged columns where needed.  
- Standardized categorical fields (city, product, segment, etc.).  

This step ensures the data is ready for analysis.

## üß© 3. Data Model Overview

- Built relationships between tables.  
- Defined primary and foreign keys.  
- Set cardinality and cross-filter direction.  
- Designed a star-schema style model.  
- Optimized the model for performance and correctness.  

The data model ensures accurate aggregations and filtering across visuals.

## ‚ùì 4. Business Questions Answered

The report is designed to answer the following questions:

1. **Top / Bottom 5 products** by:
   - Sales  
   - Profit  
   - Quantity Sold  

2. **Sales trends over time**:
   - Daily  
   - Monthly  
   - Quarterly  
   - Annually  

3. **Relationship between Sales & Profit** using comparative visuals.

4. **Average discount offered** in each discount category.

5. **Sales by different cities** using geographical or bar visuals.

6. **Total number of orders** in the selected period.

7. **Comparison of Sales / Profit / Quantity Sold** between any two user-selected periods.

8. **Detailed order-level analysis** showing:
   - Sales  
   - Profit  
   - Discount  
   - Net Sales  
   - Other related fields  

   With filters for:
   - Product  
   - Date  
   - Customer ID  
   - Promotion Categories
